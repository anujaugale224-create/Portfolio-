Module Overview

A structured 7-day journey through the fundamentals of web development — from basic HTML tags to building a complete To-Do List project using JavaScript. Click on any day to view notes, topics covered, and assignment details.

Day 1
Basic HTML Tags
p, b, i, u, marquee, headings, br, hr & more
▼

Day 1 introduced us to the building blocks of every webpage — HTML tags. We learned how to structure text content, apply basic formatting, and understand how a browser renders HTML elements.

  • HTML document structure: <!DOCTYPE html>, <html>, <head>, <body>
  • Heading tags: <h1> to <h6> — hierarchy and usage
  • Paragraph tag <p>, line break <br>, horizontal rule <hr>
  • Text formatting: <b> (bold), <i> (italic), <u> (underline), <strong>, <em>
  • Marquee tag <marquee> — scrolling text, direction, behavior attributes
  • Comments in HTML: <!-- comment -->
  • Special characters / HTML entities: &amp;, &lt;, &gt;, &nbsp;
Output
📝

Assignment: Create an HTML page about yourself using all basic tags — headings, paragraphs, bold/italic text, a marquee with your name, and a horizontal rule dividing sections.

Day 2
Media & Tables
Image, Anchor, Video/Audio, Tables
▼

Day 2 focused on embedding media and linking pages together. We also learned how to display structured data using HTML tables — one of the most versatile layout tools in basic HTML.

  • Image tag <img> — src, alt, width, height attributes
  • Anchor tag <a> — href, target (_blank, _self), linking to sections with #id
  • Video tag <video> — src, controls, autoplay, loop, muted, width
  • Audio tag <audio> — src, controls, autoplay, loop
  • Table structure: <table>, <tr>, <th>, <td>, <caption>
  • Table attributes: border, cellpadding, cellspacing, colspan, rowspan
  • Grouping: <thead>, <tbody>, <tfoot>
Output
📝

Assignment: Build a page with your photo (img tag), a link to your favourite website, an embedded video, and a table showing your semester subjects and marks.

Day 3
HTML Forms
Input types, select, textarea, form attributes
▼

Day 3 was all about user interaction — HTML forms allow websites to collect data from users. We explored all major input types and form controls used in real-world websites.

  • Form tag <form> — action, method (GET/POST) attributes
  • Input types: text, password, email, number, date, file, checkbox, radio, submit, reset, button
  • Label tag <label> and its importance for accessibility (for attribute)
  • Textarea <textarea> — rows, cols attributes
  • Select dropdown <select> and <option> tags
  • Fieldset and legend for grouping form elements
  • Required, placeholder, value, name attributes
Output
📝

Assignment: Create a student registration form with fields for name, email, phone, branch (dropdown), gender (radio), interests (checkboxes), and a submit button.

Day 4
Header, Nav, Footer & CSS Selectors
Semantic tags + all CSS selector types
▼

Day 4 introduced semantic HTML5 elements and a deep dive into CSS — the language that styles web pages. We covered every major CSS selector type used in professional development.

  • Semantic HTML5: <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>
  • CSS linking methods: inline, internal (<style>), external stylesheet
  • Element selector: p { } — targets all elements of that type
  • Class selector: .classname { } — reusable styles
  • ID selector: #idname { } — unique element styling
  • Universal selector * { }, Group selector h1, h2, p { }
  • Descendant div p, Child div > p, Adjacent h1 + p, Sibling h1 ~ p
  • Pseudo-classes: :hover, :focus, :nth-child(), :first-child, :last-child
  • Pseudo-elements: ::before, ::after, ::placeholder
  • Attribute selector: input[type="text"]
Output
📝

Assignment: Build a complete webpage layout with a styled header (logo + nav links), a main content section, and a footer — using at least 5 different CSS selector types.

Day 5
Box Model & Flexbox
margin, padding, border + CSS Flexbox layout
▼

Day 5 unlocked layout superpowers — understanding the CSS Box Model is fundamental to spacing, while Flexbox revolutionizes how we arrange elements on a page with ease and precision.

  • Box Model: content → padding → border → margin (layers explained)
  • width, height, min-width, max-width, box-sizing: border-box
  • Margin shorthand: top right bottom left; margin: auto for centering
  • Border: border-width, border-style, border-color, border-radius
  • Flexbox container: display: flex, flex-direction, flex-wrap, justify-content, align-items, align-content, gap
  • Flex items: flex-grow, flex-shrink, flex-basis, align-self, order
  • Common patterns: navbar, card row, centered content, space-between layout
Output
📝

Assignment: Create a responsive card layout using Flexbox — display 4 cards in a row that wrap on smaller screens. Each card should demonstrate proper use of the Box Model (padding, border, margin).

Day 6
JavaScript Basics & All Operations
Variables, data types, operators, functions, DOM
▼

Day 6 was a big leap — JavaScript brings pages to life! We covered the language fundamentals and all operation types, from arithmetic to DOM manipulation, which is the foundation of interactive web apps.

  • Variables: var, let, const — differences and when to use each
  • Data types: String, Number, Boolean, null, undefined, Array, Object
  • Arithmetic operators: +, -, *, /, %, ** (exponent)
  • Assignment operators: =, +=, -=, *=, /=, %=
  • Comparison operators: ==, ===, !=, !==, >, <, >=, <=
  • Logical operators: && (AND), || (OR), ! (NOT)
  • String operators & template literals: `Hello ${name}!`
  • Conditional statements: if / else if / else, ternary operator
  • Loops: for, while, do-while, for...of, for...in
  • Functions: declaration, expression, arrow functions
  • DOM manipulation: getElementById, querySelector, innerHTML, addEventListener
Output
📝

Assignment: Build a simple calculator page using HTML + CSS + JavaScript that performs all arithmetic operations (add, subtract, multiply, divide) with input fields and displays the result dynamically.

Day 7
To-Do List Project
Final project combining HTML + CSS + JavaScript
▼

Day 7 was our capstone mini-project — building a fully functional To-Do List app from scratch. This project brought together everything we had learned: semantic HTML structure, CSS Flexbox styling, and JavaScript DOM manipulation with events and local storage.

  • Project structure: HTML layout with input, button, and task list container
  • CSS styling: clean UI with Flexbox, hover effects, completed task strikethrough
  • JavaScript — adding tasks: push to array, render to DOM using innerHTML
  • Marking tasks complete: toggle class on click using classList.toggle()
  • Deleting tasks: removeChild() or filter() and re-render
  • Storing tasks: localStorage.setItem() / getItem() for persistence
  • Edge cases: empty input validation, Enter key event listener
Core JavaScript Logic Output
📝

Assignment: Submit your complete To-Do List app — must include: add task, mark complete (strikethrough), delete task, and localStorage persistence so tasks survive a page refresh.
